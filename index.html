ASP.NET Core 3.0 项目实战之订单采集小程序 第二章 需求分析

发表于 2019-09-29 分类于 .NET Core 本文字数： 1.2k 阅读时长 ≈ 2 分钟

做任何项目前，无论大小，都应该养成需求分析的习惯。这个分析不一定要非常细致，但要尽量全面。

为什么说不要求非常细致？

因为在实际的开发中，需求总是会变化。细致的需求分析不仅耽误时间，也会陷入钻牛角的情况，若以后需求有了变化，那样不仅白白做了工，很多东西或许还要推倒重来。但也不能因此什么都不做，我见过许多初中级开发人员，拿到一纸需求后，追求效率，略微在大脑里构想一遍，就开始动手 coding 了。
